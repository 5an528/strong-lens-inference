# WHY: BayesFlow 2.x needs exactly one Keras 3 backend. We standardize on
# "torch" -- it needs no separate CUDA toolkit install (the pip wheel below
# bundles its own CUDA runtime) and the Keras 3 torch backend automatically
# runs on the GPU whenever `torch.cuda.is_available()` is True, with no code
# changes needed elsewhere in the project.

# Extra (not replacement) package index so `pip install -r requirements.txt`
# can find the CUDA-enabled torch wheels below alongside everything else on
# PyPI. cu130 (CUDA 13.0) wheels are used to match this project's current
# torch/torchvision versions (see the pinned "+cu130" local version below --
# NOTE: plain "torch"/"torchvision" with no version pin is NOT enough to get
# the GPU build here. pip resolves package versions across ALL indexes
# together and picks the highest version number it finds; if the CUDA index
# only has older releases than plain PyPI (which only ships CPU wheels), pip
# silently installs the newer CPU-only one instead of erroring. Pinning the
# exact "+cu130" local version forces pip to fetch from the CUDA index,
# specifically. If you don't have an NVIDIA GPU, or want a smaller CPU-only
# install, replace the line below with the CPU index and drop "+cu130" from
# the torch/torchvision lines:
#   --extra-index-url https://download.pytorch.org/whl/cpu
--extra-index-url https://download.pytorch.org/whl/cu130
